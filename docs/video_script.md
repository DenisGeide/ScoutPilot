# Сценарий видео для интервью

Цель ролика — за 3–5 минут показать не заранее записанный сценарий, а обычный запуск Scout Pilot через автономный runtime. Основной пример — поиск вакансий. Live HH.ru может меняться или показывать блокеры, поэтому перед записью сначала сделайте пробный запуск. Для надежного дубля держите готовым локальный demo.

## Что подготовить до записи

1. Закройте окна с ключами, `.env`, личными письмами и другими приватными данными.
2. Отключите уведомления Windows и мессенджеров.
3. Откройте репозиторий на GitHub и локальный терминал.
4. Увеличьте шрифт терминала так, чтобы текст читался в видео.
5. Не показывайте ввод пароля или API-ключа. Для основного режима отдельный API-ключ не нужен: используется Codex CLI с авторизацией через ChatGPT.
6. Закройте старое окно Chromium перед запуском, чтобы применился новый размер viewport.
7. Не растягивайте Chromium через Windows Snap на всю высоту большого монитора. Оставьте запущенный размер окна и расположите терминал слева, браузер справа.

Проверка перед записью:

```bat
cd /d "C:\Users\Unknown\Desktop\AI-агент для автоматизации браузера"
codex login status
scout-pilot doctor
git status --short
```

Ожидается:

- `codex login status` сообщает `Logged in using ChatGPT`;
- `doctor` не показывает блокирующих ошибок;
- Git working tree чистый;
- Chromium установлен и проходит headless smoke.

Если нужен отдельный ручной smoke браузера:

```bat
scout-pilot browser-smoke --headless --hold-seconds 0
```

## Раскладка экрана

- Терминал занимает левую половину экрана.
- Chromium занимает правую половину, но не растянут на всю высоту монитора.
- Записывайте в 1920×1080 или обрежьте область захвата до терминала и браузера.
- Не открывайте `.env` в кадре.
- На HH.ru лучше использовать отдельный профиль без личных резюме и уведомлений.

Белая область снизу на HH.ru — это фон самой страницы после видимого списка, а не ошибка Playwright. Scout Pilot теперь запускает headed Chromium с viewport `1000×900`. Если окно вручную максимизировать или растянуть на большой монитор, пустая область снова станет заметной. Для другого размера можно локально изменить:

```env
SCOUT_PILOT_BROWSER_VIEWPORT_WIDTH=1000
SCOUT_PILOT_BROWSER_VIEWPORT_HEIGHT=900
```

Не уменьшайте высоту ниже `900` для основного demo: часть карточек может оказаться вне видимой semantic observation.

## Сценарий на 3–5 минут

### 0:00–0:20 — приветствие

Скажите:

> Привет. Это Scout Pilot — автономный браузерный агент на Python и Playwright. Я передаю ему задачу обычным текстом, а дальше он сам наблюдает страницу, выбирает семантические инструменты и выполняет многошаговый сценарий. Сейчас покажу живой запуск, управление контекстом и остановку перед внешним действием.

Не называйте проект production-ready. Это интервью-проект с детерминированными тестами и ручным live smoke.

### 0:20–0:45 — коротко показать репозиторий

Откройте README и на несколько секунд покажите структуру `src/scout_pilot`.

Скажите:

> Playwright изолирован в Browser Engine. Остальной код не получает Page, DOM или полный HTML. Semantic Observation Engine отдает компактные секции, поля и интерактивные элементы со стабильными ID. Tool Runtime выполняет действия только после Security Policy. LLM-провайдер можно менять без изменения browser tools.

Не листайте каждый файл. Достаточно показать папки `browser`, `observation`, `tools`, `llm`, `runtime`, `security`, `context` и `memory`.

### 0:45–1:05 — проверка окружения

В терминале выполните:

```bat
codex login status
scout-pilot doctor
```

Скажите:

> В этом запуске reasoning идет через локальный Codex CLI, авторизованный через ChatGPT. Ключ из `.env` не используется. Автоматические тесты не вызывают платные модели и не ходят на HH.ru.

### 1:05–1:25 — запуск агента

Выполните:

```bat
scout-pilot menu
```

Введите:

```text
0
```

На вопрос про URL введите:

```text
https://hh.ru
```

Дождитесь появления Chromium. Не максимизируйте окно.

Скажите:

> Браузер видимый и использует persistent profile. Внутри диалога можно давать несколько задач подряд, не перезапуская Chromium.

### 1:25–3:20 — основной сложный запрос

Для основного дубля используйте **Промпт 1**. Вставьте его целиком:

```text
Найди три разные вакансии AI Engineer или Python AI Developer с зарплатой от 250 000 рублей. Открой и прочитай каждую вакансию отдельно. Для каждой укажи название, компанию, зарплату, опыт, основные требования и точную ссылку. Не открывай одну вакансию повторно. Не откликайся и ничего не отправляй.
```

Пока агент работает, коротко комментируйте только наблюдаемое:

> Агент нашел поле поиска через роль и доступное имя, а не через CSS-селектор. Сейчас он получает ограниченное semantic observation. Повторная шапка и служебные элементы удаляются, а карточки результатов и ссылки получают приоритет. Значения полей в отчет не попадают.

Не говорите, что результат гарантирован заранее. Если найдено меньше трех точных вариантов, честное объяснение ограничений выглядит лучше выдуманного ответа.

В итоговом ответе проверьте:

- название вакансии;
- зарплату и опыт;
- краткие требования;
- точный URL под вакансией или в разделе `Ссылки`;
- фразу о том, что отклик не отправлялся.

После ответа введите `/open 1`. Первая точная ссылка должна открыться в том же окне
Chromium. Это надежный вариант для старого `cmd.exe`; в Windows Terminal дополнительно
работает `Ctrl + клик`.

### 3:20–4:05 — follow-up без перезапуска

В той же строке `Вы >` отправьте:

```text
Из уже найденных вариантов выбери два наиболее подходящих для Python-разработчика с опытом LLM-приложений. Сравни их в компактной таблице по Python, LLM, RAG, MLOps, английскому языку, формату работы и рискам несоответствия. Используй только данные, которые реально видны на страницах, сохрани ссылки и не придумывай отсутствующие требования.
```

Скажите:

> Это продолжение той же сессии. Runtime использует task memory и компактные episodic summaries, а не пересылает всю историю и полный HTML на каждом шаге.

### 4:05–4:35 — показать Security Policy

Только для демонстрации границы безопасности отправьте:

```text
Нажми кнопку «Откликнуться» у лучшей вакансии.
```

Когда появится подтверждение, ничего не подтверждайте. Ответьте:

```text
нет
```

Скажите:

> Модель предложила действие, но не может сама разрешить его. Детерминированная Security Policy остановила Tool Runtime до клика. Навигация и поиск проходят автоматически, а отклик, сообщение, удаление и оплата требуют решения пользователя.

### 4:35–5:00 — отчет и завершение

Введите:

```text
/report
```

Ссылки в итоговом ответе пронумерованы. В Windows Terminal их можно открыть через
`Ctrl + клик`; в старом `cmd.exe` введите `/open 1`, чтобы Scout Pilot открыл точный URL
в уже запущенном браузере агента.

Затем:

```text
/exit
```

Скажите:

> Report и replay сохраняют наблюдения, решения, инструменты, метрики контекста и security pauses. В них нет raw HTML, cookies, токенов, профиля браузера или значений чувствительных форм. Live HH.ru остается ручным smoke, а основная автоматическая проверка работает на локальных синтетических страницах.

## Три готовых сложных промпта

Для основного ролика достаточно **Промпта 1** и короткого follow-up выше. Промпты 2 и 3
используйте для запасных дублей или отдельной демонстрации. Не запускайте все три подряд в
одном пятиминутном видео.

### Промпт 1 — многостраничный поиск

```text
Найди три разные вакансии AI Engineer или Python AI Developer с зарплатой от 250 000 рублей. Открой и прочитай каждую вакансию отдельно. Для каждой укажи название, компанию, зарплату, опыт, основные требования и точную ссылку. Не открывай одну вакансию повторно. Не откликайся и ничего не отправляй.
```

Что демонстрирует: поиск, переходы между страницами, защиту от повторного URL, чтение
длинного описания ниже viewport, task memory и точные ссылки.

### Промпт 2 — строгая фильтрация и доказательства

```text
Найди три разные вакансии AI Engineer, LLM Engineer или Python AI Developer с зарплатой от 250 000 рублей у разных работодателей. Открой и прочитай каждую отдельно. Исключи робототехнику, обычный backend и вакансии, где AI упомянут только в названии продукта. Для каждого варианта укажи название, компанию, зарплату, опыт, формат работы, основной стек, три ключевых требования, короткое доказательство связи с AI/ML/LLM и точную ссылку. Если точных вариантов меньше трех, честно укажи, какой критерий не выполнен. Не откликайся и ничего не отправляй.
```

Что демонстрирует: работу с ограничениями, отсев нерелевантных результатов, извлечение
фактов из описаний и честную обработку недостатка данных.

### Промпт 3 — техническое сравнение и рейтинг

```text
Найди три разные вакансии, связанные с разработкой LLM-приложений на Python, с указанной зарплатой от 250 000 рублей. Открой каждую страницу отдельно и не возвращайся к уже прочитанной вакансии. Для каждой выпиши компанию, зарплату, опыт, обязанности, обязательные и желательные требования, технологии Python, FastAPI, LLM, RAG, Docker, базы данных и точную ссылку. Затем оцени соответствие кандидату со стеком Python, FastAPI, PostgreSQL, Docker и RAG по шкале от 0 до 10. Каждый балл обоснуй только фактами со страницы; отсутствующие данные помечай «не указано». Не откликайся и ничего не отправляй.
```

Что демонстрирует: многошаговый анализ, разделение обязательных и желательных требований,
сравнение нескольких источников и отказ от выдуманных фактов.

## Резервный demo, если HH.ru нестабилен

Если HH.ru показывает CAPTCHA, login wall или меняет разметку, не пытайтесь обходить блокировку. Скажите, что live-сайт является ручным smoke, и запустите детерминированный сценарий:

```bat
scout-pilot live-local-demo --headed --slow-mo-ms 120 --dashboard compact
```

Этот demo проходит через настоящий runtime, читает три локальные страницы и останавливается перед `Apply`. Он не вызывает внешний LLM и не подделывает live HH.ru результат.

## Что не нужно говорить

- Не называйте проект промышленной системой или готовым SaaS.
- Не утверждайте, что HH.ru всегда работает одинаково.
- Не говорите, что агент видит полный DOM или HTML.
- Не обещайте стопроцентную точность LLM.
- Не показывайте API-ключи, `.env`, cookies или данные browser profile.
- Не выдавайте локальный mock demo за live-провайдера.
- Не подтверждайте реальный отклик во время записи.

## Короткое завершение

Финальная фраза:

> Главная часть проекта для меня — не один сценарий под HH.ru, а разделение ответственности: безопасное semantic observation, provider-neutral reasoning и tools, ограниченный контекст, восстановление после ошибок и независимая политика внешних действий. Ограничения live-сайта я не скрываю, поэтому рядом есть воспроизводимый локальный demo и детерминированные тесты.
